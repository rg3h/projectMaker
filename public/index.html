Description

projectMaker is a command-line tool that creates new projects quickly and easily.

> projectMaker   (fully interactive)
or
> projectMaker -p newProject -t someTemplate.zip

projectMaker copies a template and replaces all {{varName}} strings with the values you provide. This is similar to mustasche or handlebars. projectMaker is written in javascript and uses node to run from the command-line, making projectMaker easy to inspect, modify, and extend.

Templates

A template is just a directory stored on github containing files with strings denoted by "{{" varName "}}"   
For example:
You might create a project template with several html files that reference the title of the project: 

  <div class=topBar>
    <div class=projectTitle>{{projectTitle}}</div>
    <div class=authors>{{authorList}}</div>
  </div>

Then you can use projectMaker to create new projects from this template, replacing the {{projectTitle}} with the new project title.

By default, projectMaker uses the template: https://github.com/rg3h/templateWebApp
I use this template to create a new web app quickly (the template even comes with a local server).

projectMaker Steps

projectMaker does the following:

  1. asks for the new project directory name (defaulting to newProject)
  2. asks for the template url with the above as the default
  3. downloads and unzips the template and moves the nested directory up one level
  4. searches through the files for {{var}} strings and stores them in a set
  5. asks for a value for each {{var}} and replaces it everywhere in the new project
  6. shows stats when done

Command Line Flags:

-h | --help shows helpful information
-p | --projectDir newProjectName (default: newProject))
-t | --templateUrl url points to a github zip holding files with {{vars}}

Installing and Running projectMaker

  1. download node.js so you can run javascript from the command line
  2. go to https://github.com/rg3h/projectMaker and fork or download the zip file
  3. unzip it into some directory (e.g. projectMaker
  4. cd projectMaker/src and run: node .

Windows Bonus

For Windows, you can make projectMaker run from any shell by doing the following:
Le't say you have a directory that is in the executable path (e.g. Program Files or C:\rg3h\bin\). And let's say you downloaded and uzipped projectMaker from github into C:\rg3h\downloads\projectMaker
  1. cd C:\rg3h\bin    (a dir your executable path)
  2. makedir projectMaker
  3. cd C:\rg3h\bin\projectMaker
  4. copy C:\rg3h\downloads\projectMaker\src .
  5. copy C:\rg3h\downloads\projectMaker\tools\windows\putMeInExePathAs_projectMaker.bat projectMaker.bat
now you should be able to run projectMaker from any shell

UnInstalling projectMaker

  1. Just delete the projectMaker directory
  2. optionally uninstall node.js if you no londer need it

Details on How projectMaker Works

All of the javascript code is in projectMaker/src

projectMaker uses node.js to run on the command line and be able to make the new project's directory

projectMaker uses only one third party node.js module yauzl (yet another unzip library) to unzip the github template.

Here is the projectMaker code in src/
config.js -- handles configuration such as the command parameters

help.js -- shows the introduction message and the help message

main.js -- main entry point for projectMaker

nodeTimer.js -- wraps node's precision timer for stats

node_modules -- 3rd party modules (yauzl)

package-lock.json -- node's version locking mechanism

package.json -- node's packaging mechanism

stats.js -- gathers timing and file stats to display when done

templateProcessor.js -- loads, unzips, and processes the template files

tty.js -- create ascii boxes and colors in the shell

util.js -- general utility functions

Another Useful Tool

In addition to projectMaker, consider using https://github.com/rg3h/gt as an easy way to manage your github projects.

For example, after using projectMaker to make myFunProject you could then:

  • cd myFunProject
  • gt addRepo .   
  • gt checkInAll "initial check-in"   to push all modified files to github
