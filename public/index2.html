Description

projectMaker is a command-line function that creates new projects quickly and easily. It copies a selected template and replaces all {{varName}} strings with the values you provide. This is similar to mustasche or handlebars. projectMaker is written in node.js javascript.

As an example, you might create a project template with several html files that reference the name of the project. You then put a {{projectTitle}} string in each of them. Later you can use projectMaker to create new projects, telling it to replace the {{projectTitle}} with the new project name.

projectMaker is simple, quick, and easy to use. Since the code is written in javascript, it is easy to inspect and extend.

Templates

A project template is just a directory stored on github containing files with special strings in them. These strings are denoted by "{{" varName "}}"    for example {{projectName}} or {{authorList}}

For each unique {{variable}} string projectMaker asks you for a replacement value. By default, projectMaker uses the template:

https://github.com/rg3h/templateWebApp
I use this template to create a new web app quickly (it even comes with a local server).

projectMaker Steps

projectMaker does the following:

  1. asks for the new project directory name (defaulting to newProject)
  2. asks for the template url (with the above as the default), downloads and unzips it
  3. searches through the files for {{var}} strings and stores them in a set
  4. asks for a value for each {{var}} and replaces it everywhere in the new project
  5. shows stats when done

Examples

projectMaker // interactively asks for the values
projectMaker -p myNewProject -t https://github/you/template/main.zip
projectMaker -h // shows help

Command Line Flags:

-p | --projectDir newProjectName (default: newProject))
-h | --help shows helpful information
-t | --templateUrl url points to a github zip holding files with {{vars}}

Installing and Running projectMaker

  1. go to https://github.com/rg3h/projectMaker and fork or download the javascript
  2. download node.js so you can run javascript from the command line
  3. cd projectMaker/src and run node .

UnInstalling projectMaker

  1. Just delete the projectMaker directory
  2. optionally uninstall node.js if you no londer need it

Details on How projectMaker Works

All of the javascript code is in projectMaker/src

projectMaker uses node.js to run on the command line and have access to the file system (for example to make the new project's directory)

projectMaker uses one third party node.js module yauzl (yet another unzip library for node) to unzip the github template.

Here is the projectMaker code in src/
config.js -- handles configuration such as the command parameters

help.js -- shows the introduction message and the help message

main.js -- main entry point for projectMaker

nodeTimer.js -- wraps node's precision timer for stats

node_modules -- 3rd party modules (yauzl)

package-lock.json -- node's version locking mechanism

package.json -- node's packaging mechanism

stats.js -- gathers timing and file stats to display when done

templateProcessor.js -- loads, unzips, and processes the template files

tty.js -- a simple way to create ascii boxes and colors

util.js -- general utility functions

Another Useful Tool

In addition to projectMaker, consider using https://github.com/rg3h/gt as an easy way to manage your github projects.

For example, after using projectMaker to make myFunProject you could then:

  • cd myFunProject
  • gt addRepo .    add myFunProject to github
  • gt checkInAll "updated files"   to push all modified files to github
